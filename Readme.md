# 🚀 STM32 Motor Control Lab (Cascade PID + Alpha Filter)

โค้ดสำหรับควบคุม DC Motor ด้วยบอร์ด STM32 (STM32G474RE) โค้ดชุดนี้ถูกออกแบบมาเพื่อแก้ปัญหามอเตอร์สั่น คราง และจูนยาก โดยใช้โครงสร้างการควบคุมแบบ **Cascade PID** (ระบบซ้อนทับ) และใช้ **Alpha Filter** กรองสัญญาณรบกวนจาก ADC

## ✨ จุดเด่นของโค้ดนี้
1. **Alpha Filter (EMA Filter):** กรองสัญญาณ Noise จาก R-เกือกม้า ทำให้เป้าหมายนิ่งสนิท มอเตอร์ไม่สั่นยุกยิกเวลาหยุดนิ่ง
2. **Cascade PID Control:** แยกการทำงานเป็น 2 ลูป 
   - **Loop นอก (500Hz):** คุมตำแหน่ง (Position) สั่งงานเป็นความเร็ว (RPM)
   - **Loop ใน (1kHz):** คุมความเร็ว (Velocity) รับคำสั่งมาแปลงเป็น PWM
   *ข้อดีคือใช้แค่ P คุมตำแหน่ง และ PI คุมความเร็ว ไม่ต้องง้อตัว D ให้ปวดหัว!*
3. **Deadband:** ป้องกันมอเตอร์ครางหึ่งๆ เวลาหมุนถึงเป้าหมายแล้วมี Error ระดับทศนิยม

## 🔌 1. การต่อสายไฟ (Wiring Diagram)

### R-เกือกม้า (Potentiometer)
* **ขาซ้าย** ➡️ `3.3V`
* **ขากลาง (Signal)** ➡️ `PC0` (ADC1_IN6)
* **ขาขวา** ➡️ `GND`

### เซนเซอร์ท้ายมอเตอร์ (Encoder/QEI)
* **VCC (สายสีดำ)** ➡️ `3.3V`
* **GND (สายสีน้ำเงิน)** ➡️ `GND`
* **Phase A / C1 (สายสีเขียว)** ➡️ `PA6` (TIM3_CH1)
* **Phase B / C2 (สายสีเหลือง)** ➡️ `PA4` (TIM3_CH2)

### บอร์ดขับมอเตอร์ (DRV8833 - บอร์ดแดง)
* **VM** ➡️ `5V` (ไฟเลี้ยงมอเตอร์)
* **GND** ➡️ `GND`
* **STBY** ➡️ `3.3V` *(ห้ามลืม! ถ้าไม่ต่อมอเตอร์จะไม่หมุน)*
* **AIN1** ➡️ `PA0` (TIM2_CH1)
* **AIN2** ➡️ `PA1` (TIM2_CH2)
* **AO1 / AO2** ➡️ ต่อเข้าสายมอเตอร์ (สีเทา/สีแดง)

## 🎮 2. วิธีใช้งานเบื้องต้น (โหมดการทำงาน)

ตัวแปรหลักที่ใช้ควบคุมคือ `TASK_MODE` (เปลี่ยนค่าได้ใน STM32CubeMonitor หรือ Live Expressions)
* `TASK_MODE = 1` : **โหมดคุมตำแหน่ง (Position Control)** บิดเกือกม้า 1 รอบ = มอเตอร์หมุน 0 - 360 องศา
* `TASK_MODE = 2` : **โหมดคุมความเร็ว (Velocity Control)** บิดเกือกม้า = มอเตอร์หมุน 0 - 120 RPM

**💡 วิธีสั่งงาน:**
ระบบจะอ่านค่าคันเร่ง **"ก็ต่อเมื่อกดปุ่ม B1 (ปุ่มสีฟ้า) ค้างไว้เท่านั้น"** ถ้าปล่อยปุ่มเมื่อไหร่ ระบบจะล็อกเป้าหมายไว้ที่ค่าล่าสุดทันที

## 🖥️ 3. การเซ็ตอัพ STM32CubeMonitor (สำหรับดูกราฟและจูนสด)

1. Build โค้ดใน STM32CubeIDE ให้ผ่าน เพื่อให้ได้ไฟล์ `.elf`
2. เปิด STM32CubeMonitor โยง Node ตามนี้:
   - **ฝั่งดูกราฟ:** `myProbe_In` ➡️ `myVariables` (เลือกไฟล์ .elf) ➡️ `Processing` ➡️ `Chart`
   - **ฝั่งเขียนค่า (Write):** ใช้โหนด `numeric` (Dashboard) ➡️ `write` ➡️ `myProbe_Out`
3. **ตัวแปรที่ต้องดึงมาดูกราฟ:**
   - `target_rpm` คู่กับ `QEIdata.QEIAngularVelocity_RPM`
   - `target_pos_deg` คู่กับ `current_pos_deg`
4. **ตัวแปรที่ต้องดึงมาทำปุ่ม Write (จูนค่า):**
   - `TASK_MODE`
   - `Kp_v` และ `Ki_v` (สำหรับวงใน - ความเร็ว)
   - `Kp_p` (สำหรับวงนอก - ตำแหน่ง)

## 🎛️ 4. คู่มือการจูน Cascade PID (Tuning Guide)

**กฎเหล็ก:** ต้องจูน "ความเร็ว (วงใน)" ให้เสร็จก่อน แล้วค่อยจูน "ตำแหน่ง (วงนอก)"

### สเตป 1: จูนความเร็ว (Velocity Loop)
1. ตั้งค่า `TASK_MODE = 2`
2. เซ็ตค่าเริ่มต้น: `Kp_v = 1.0`, `Ki_v = 0.0`
3. กดปุ่ม B1 ค้าง แล้วบิดเกือกม้าให้ `target_rpm` อยู่ที่ประมาณ 60
4. **จูน Kp_v:** ค่อยๆ เพิ่มทีละ 0.5 จนมอเตอร์หมุนได้ความเร็วใกล้ 60 (ถ้าสั่นครางให้ลดลง)
5. **จูน Ki_v:** ค่อยๆ เพิ่มทีละ 1.0 เพื่อให้ตัว I ดันความเร็วจริงให้ไปแตะเส้น 60 RPM พอดีเป๊ะ (ถ้าเส้นกราฟสวิงเป็นคลื่นแปลว่า Ki มากไป ให้ลดลง)

### สเตป 2: จูนตำแหน่ง (Position Loop)
1. ตั้งค่า `TASK_MODE = 1`
2. เซ็ตค่าเริ่มต้น: `Kp_p = 1.0` (ไม่ต้องใช้ Ki_p และ Kd_p)
3. กดปุ่ม B1 ค้าง แล้วบิดเกือกม้าให้เป้าหมายไปที่ 180 องศา
4. **จูน Kp_p:** - ถ้ามอเตอร์วิ่งไปหาเป้าหมายช้า (อืด) ให้ **เพิ่ม** `Kp_p` (เช่น เป็น 2.0, 3.0)
   - ถ้ามอเตอร์วิ่งเร็วไปจนเลยเป้าหมายแล้วตีกลับ (Overshoot) ให้ **ลด** `Kp_p` ลง
